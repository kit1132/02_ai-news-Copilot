## 最優先: Claude Fable 5 / Mythos 5 が全世界で停止

### 米政府の輸出管理指示で、Anthropicが両モデルを即時無効化 (6/12)
公開からわずか3日で最上位モデルが利用不能に。M365 Copilot / GitHub Copilot / Foundry / Devin / Claude Code など、Anthropicの最新モデルに依存する全レイヤーに波及。

- **経緯**: 6/12 17:21 ET着の指示書で、米商務省（BIS、Lutnick長官名）が国家安全保障・輸出管理を理由に「外国籍ユーザーへのFable 5 / Mythos 5アクセス停止」を要求。Anthropicは外国籍をリアルタイムに識別・除外できないため、**有料エンタープライズ・自社従業員を含む全ユーザー向けに両モデルを全世界で停止**して指示に対応
- **挙動**: 進行中のFable 5 / Mythos 5セッションはエラー終了し、新規クエリは旧Opus 4.8など下位モデルへ自動ルーティング。Opus 4.8 / Sonnet / Haikuは影響なし
- **Anthropicの公式声明**: 「これは誤解だと考えており、可能な限り早期のアクセス復旧に取り組む」と表明し顧客に謝罪。背景にはFable 5のジェイルブレイクによるコード脆弱性解析手法への懸念があると報じられるが、指示書は具体的な国家安全保障上の懸念を明示せず

### 開発ツール・Microsoftレイヤーへの波及
- **Devin**: 6/12にFable 5を製品から削除（Opus 4.8 / GPT-5.5は継続提供）
- **Claude Code**: v2.1.170以降で提供していたFable（`/model fable`）が利用不可に。v2.1.174で入れた「Fable 5クレジット消費バナー」関連の修正が、今や提供停止で実質無効化
- **Microsoft（Foundry / GitHub Copilot）**: 外部顧客はFoundry・GitHub Copilot経由でFable 5を利用できていたが今回の停止で不可。なおMicrosoftは社内では既にFable 5を遮断済み（Anthropicの30日〜最大2年のデータ保持ポリシーを法務がレビュー中のため）。Foundry提携・Copilot Cowork等の協業自体は継続
- https://winbuzzer.com/2026/05/15/microsoft-starts-canceling-claude-code-licenses-xcxwbn/

### エンタープライズ向け実務メモ
- Fable 5 / Mythos 5を本番ワークフローに組み込んでいた組織は、当面Opus 4.8 / GPT-5.5へのフォールバックを前提に運用すること。復旧時期は未定で「誤解の解消次第」と流動的
- AIモデル供給が政府指示で一夜にして停止しうるリスクが顕在化。単一モデル依存を避け、フォールバック先を事前に定義しておく重要性が再確認された
