# 出力スタイル

## 言語

- 日本語

## フォーマット

- カテゴリ分類 + 要約
- 各項目2-3行で簡潔に
- 全体で5-15項目が目安
- ソースURLは必ず含める

## トーン

- 冗長な前置き不要
- 事実ベース、感想は不要
- 重要度が高いものほど上に配置
- 情報が少ない日は無理に増やさない（「特筆事項なし」でOK）
- 長すぎる説明・お世辞・精神論は不要

## ファイル形式

- Markdownファイルとして保存
- パス: `digests/YYYY/MM/ai-news-YYYY-MM-DD.md`

## 改善メモ

- ダイジェスト末尾に「改善メモ」セクションを設ける
- 内容: ソースURL変更、新規監視対象の提案、取得方法の改善等
- 同じ提案が2回以上出たら、rulesファイルへの反映を強く推奨
- **フォールバック発生時の記録**: 取得に失敗したソースがある場合、改善メモに以下を記載する
  - 失敗URL・ステータスコード（例: `anthropic.com/news → 403`）
  - 最終的に成功した方法（代替URL / WebSearch / 取得不可）
  - 発生日（継続的な403は初回のみ記録し、以降は省略可）
