## Claude Code

### v2.1.152（5/27）
33件の変更を含むリリース。

- **`/code-review --fix`**: レビュー結果をワーキングツリーに自動適用。再利用・簡略化・効率改善の提案をそのまま反映
- **`/simplify`**: `/code-review --fix` のエイリアスとして動作
- **`/reload-skills`**: セッション再起動なしでスキルディレクトリを再スキャン
- **Skills `disallowed-tools`**: フロントマターで特定ツールをスキル実行中に除外可能
- **`SessionStart` hooks**: `reloadSkills: true` を返すことでスキル再読み込みをトリガー
- **`MessageDisplay` hook**: アシスタントメッセージテキストの変換・非表示が可能
- **バグ修正**: 長時間セッションのターミナルスタイル劣化、同一コマンド/異なる環境変数のMCPサーバー重複排除、リモートMCP接続（egress proxy有効時）、モデル切替後のthinking-block署名の陳腐化
- https://github.com/anthropics/claude-code/releases

## GitHub Copilot

### Copilot Memory 管理強化（5/26 Changelog）
Copilot Memory（Public Preview）に新しい制御機能を追加。

- **リポジトリ単位の無効化**: リポジトリ管理者がSettings > Copilot機能設定からMemoryをオフ可能。リポジトリレベルのfact保存・読み取りを完全停止
- **CLI `/memory` コマンド**: `/memory on`・`/memory off`・`/memory show` でセッション横断的にMemoryを管理
- **削除ガイダンス改善**: 「忘れて」リクエスト時に正しい削除先を案内。memory投票のダウンボートにも対応
- **権限プロンプト明確化**: memory保存時にユーザーレベル（個人設定）かリポジトリレベル（全コントリビューター共有）かを明示
- https://github.blog/changelog/2026-05-26-copilot-memory-has-more-controls-for-deletion-scope-and-the-copilot-cli/

### Copilot Chat on Web: モデル選択肢変更（5/20）
github.com上のCopilot Chat利用可能モデルを整理。

- **削除**: Gemini全モデル、GPT-5.2 Codex、GPT-5.4 nano
- **残存**: OpenAI系・Claude系モデルは全プランで引き続き利用可能
- 一貫した応答品質の確保が目的
- https://github.blog/changelog/2026-05-20-updates-to-available-models-in-copilot-on-web/
