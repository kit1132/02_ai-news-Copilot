## AI開発ツール

### Claude Code v2.1.199（7/2）・v2.1.200（7/3） — 7/3で取りこぼした2版を回収
[7/2ダイジェスト](./ai-news-2026-07-02.md)の v2.1.198 以降、2版がリリースされていた（前回状態は「7/2〜7/3 新規なし」と誤記録）。

- **v2.1.200（7/3）**: 既定パーミッションモード「default」を **「Manual」に改称**（CLI / --help / VS Code / JetBrains 横断。`--permission-mode manual` と `"defaultMode": "manual"` を受理）。AskUserQuestion ダイアログが**既定で自動継続しなくなった**（`/config` でアイドルタイムアウトをオプトイン可）。`.claude.json` の disabledMcpServers/enabledMcpServers が非配列だと起動クラッシュ→修正。スリープ復帰後にバックグラウンドセッションが無言停止する不具合→修正
- **v2.1.199（7/2）**: スタックしたスラッシュスキル（`/a /b`）を**先頭5件までロード**（従来1件）。SSL証明書エラーは即時失敗＋対処ガイド表示。ストリーミング途中のAPIエラーで**部分出力を破棄せず保持**、レート制限で打ち切られたサブエージェントも部分成果を返す
- https://github.com/anthropics/claude-code/releases

- https://github.blog/changelog/2026-07-02-copilot-cli-no-longer-needs-a-personal-access-token-in-github-actions/

### 【破壊的/セキュリティ】Cursor に RCE 脆弱性「DuneSlide」開示 — 3.0 未満は要更新
Cato AI Labs が Cursor の2件の重大脆弱性を開示（~7/2 報道）。**間接プロンプトインジェクション → サンドボックス脱出 → 開発者マシン上でのコマンド実行（RCE）**。

- **CVE-2026-50548 / CVE-2026-50549**（いずれも CVSS 3.1 で 9.8）。攻撃経路は MCP 接続先や Web 検索が返すページに仕込まれた指示文（利用者はタイプしない）
- 50548 は `run_terminal_cmd` の `working_directory` を非既定パスにすると無検証で書き込み許可に追加される点を悪用、サンドボックスヘルパー自体を上書きして以降のコマンドを無サンドボックス化。50549 はシンボリックリンク解決失敗時にフォールバックで in-project パスを信用してしまう欠陥
- **Cursor 3.0（4/2）で修正済み。3.0 未満は全バージョン影響**。実攻撃の確認はなし（研究開示）。→ **対応: 組織の Cursor を 3.0 以上へ**
- https://thehackernews.com/2026/07/critical-cursor-flaws-could-let-prompt.html

### Devin Security Swarm 提供開始 — エンタープライズ向けの脆弱性発見・修正エージェント
Cognition が **Devin Security Swarm** を全世界のエンタープライズ顧客向けに提供開始（今週）。エージェント型 map-reduce アーキテクチャで並列エージェントが脆弱性を探索し、**各検出を隔離サンドボックスで実行時に再現して悪用可能性を確認 → パッチ自動生成 → PR 起票**まで行う。本日は Cursor の RCE 開示と合わせ、AI 開発ツールのセキュリティが2件揃った格好。
- https://www.prnewswire.com/news-releases/cognition-launches-devin-security-swarm-to-tackle-the-vulnerability-backlog-302814800.html
